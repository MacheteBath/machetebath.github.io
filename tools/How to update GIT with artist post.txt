1. Run the Python Script
Open your terminal in your website tools directory and run your script.

Command: python3 extract_sc.py

Input: The script will prompt for the embed code link to extract the information.

Automation: The script will automatically generate the correctly formatted .md file with the current date and place it in the _artists/ directory.

2. Verify the New File
Check the _artists/ folder to ensure the file was created correctly. It should look like this:

Markdown
---
name: "Artist Name"
track_title: "Song Title"
sc_url: "https://api.soundcloud.com/tracks/123456789"
date: 2026-04-17
---
The artist's bio text is here...



3. Git Workflow (Safe Update)
To ensure everything is working and your main branch stays stable, use a temporary branch:

Create a new branch:
 

Stage and Commit:
git add .
git commit -m "Add [Artist Name] to spotlight via script"

Push and Merge:
git push origin feature/add-artist
Go to GitHub, open a Pull Request, and merge it into main.

4. Test Locally
Before pushing to GitHub, make sure it looks right:

	1. Open your terminal in the project folder.

	2. Run: bundle exec jekyll serve

	3. Go to http://127.0.0.1:4000/ in your browser and check the Spotlight page.


3. Merge into Main
Now that your work is "safe" on the branch, you need to move it into the main branch, which is what GitHub Pages actually displays to the world.

Bash
# Switch your local view back to the main branch
git checkout main

# Pull any changes that might have happened on GitHub while you were working
git pull origin main

# Combine the work from your feature branch into main
git merge feature/add-artist

# Upload the updated main branch to GitHub to trigger the website update
git push origin main
4. Cleanup
Once the merge is successful and you see your website update, you can delete the temporary branch to keep your workspace clean.

Bash
# Delete the local branch
git branch -d feature/add-artist

# Prune your list so it matches GitHub's clean state
git fetch -p
Pro-Tip for Git Bash
If you ever get "lost" and aren't sure which branch you are on, just type:
git status
5. Confirm on the Site
Wait about 30–60 seconds for GitHub Actions to finish building, then refresh machetebath.github.io/artists to see the new spotlight entry at the top of the list.
